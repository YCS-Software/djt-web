> Agent ID: `ad8ef6c098ab38628`
> Task: Redesign the DJT EV admin login (email + password, no OTP) with an EV-charging / station-map / QR-payment themed professional background.
> Files owned: `frontend/src/components/layout/AuthLayout.tsx`, `frontend/src/pages/auth/Login.tsx`

## Design rationale
The admin login was a single centered card on a plain grey background — functional but generic. For a production EV charging platform, the login is the brand's first impression, so it was rebuilt as a **split-screen**: an immersive branded story on the left, a focused, low-friction sign-in card on the right. This pattern is the de-facto standard for modern SaaS/admin consoles (Stripe, Vercel, Linear) because it conveys product identity while keeping the form clean and conversion-focused. All visual storytelling is done with pure CSS gradients + inline SVG, so there are zero network/image dependencies and no new npm packages.

## Layout structure
- **Root**: full-height (`100vh`) flex row.
- **Left panel (`flex 52–58%`, `display none` on `xs`/`sm`, visible `md+`)**: vertically distributed into three zones — brand lockup (top), centerpiece SVG scene + headline (middle), feature highlights row (bottom). Two absolutely-positioned decorative layers sit behind content: a masked circuit/grid overlay and a radial accent glow.
- **Right panel (`flex 1`)**: centered white `Paper` card (`maxWidth 420`, soft shadow + hairline border, `elevation={0}`) wrapping the unchanged `<Outlet/>`. Includes a **compact brand lockup shown only on `xs`/`sm`** so mobile users still see "DJT EV" once the left panel is hidden.
- **Login form**: heading "Welcome back", subtext "Sign in to the DJT EV admin console", email + password fields with adornment icons, show/hide password toggle, a strong primary "Sign In" button, and an "Authorized administrators only" footnote.

## Color palette
- Theme primary green `#4CAF50` (dark `#388E3C`) — buttons, brand chip, primary text.
- Left-panel gradient: deep green → teal → emerald `#06241b → #0b3d2e → #0a5c46` (135°).
- EV accent brights: `#00C853` and `#69F0AE` (bolt, pins, glow, brand chip gradient).
- Surface text on panel: `#E8F5E9` with reduced-opacity variants for hierarchy.
- Card surface: white on `background.default` grey, divider-colored border.

## EV / QR / map visual concept (inline SVG)
A single `viewBox="0 0 400 300"` SVG renders a stylized **map card** (dark gradient + teal stroke) containing: dashed curved **routes/roads**, two **electric-station location pins** with radial glows, a circular **charging-bolt badge** (gradient lightning glyph), and a white **QR-code square** built from rects to represent wallet/amount transfers. SVG `defs` hold reusable linear/radial gradients. The scene carries a `role="img"` + descriptive `aria-label`. Feature highlights use MUI icons `LocationOn`, `QrCode2`, `BoltOutlined`; brand chips use `EvStation`.

## Responsive behavior
- `xs`/`sm`: left panel `display: none`; right card spans available width with reduced padding (`p: 2→3`), compact brand shown atop the form. Fully usable single-column.
- `md`: split ~52/48 with `p: 5`.
- `lg+`: split ~58/42 with `p: 7` for a more cinematic panel.
- SVG scales fluidly (`width: 100%`, `maxWidth: 460`, `height: auto`). Feature row wraps via `flexWrap`.

## Accessibility
Decorative layers marked `aria-hidden`; SVG labeled; form uses `noValidate` (validation handled by formik/Yup, preserved), semantic headings, retained field labels and adornments, password visibility toggle intact, focus-visible defaults from MUI.
