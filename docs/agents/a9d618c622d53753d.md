## Design rationale
The redesign turns a functional-but-flat panel into a polished Materially/Berry-style analytics page for the **DJT EV** charging network. Goals: crisp KPI hierarchy, a data-rich primary chart, a glanceable station-health donut, and restyled list cards with real visual hierarchy (avatars, progress bars, ranked badges, empty states). The EV domain is reflected throughout — kWh energy, ₹ revenue, QR/wallet top-ups captions, connector counts, and a green-forward palette.

## Layout / grid structure
All widgets sit in MUI `Grid` containers with `spacing={3}`:

1. **Header** — `PageHeader` (title "Dashboard" + network-overview subtitle).
2. **KPI row** — 4× `StatsCard`, each `Grid item xs={12} sm={6} md={3}` (1 col mobile → 2 col tablet → 4 col desktop).
3. **Charts row** — Session Trends `md={8}` + Station Status donut `md={4}` (stack to full width on `xs`). Both cards `height: '100%'` so they align.
4. **Lists row** — Live Sessions `md={6}` + Top Stations `md={6}` (stack on `xs`).

## Chart choices & color palette
- **Session Trends**: smooth gradient **area chart** (`react-apexcharts`, `type="area"`), dual series — Sessions and Energy (kWh) — over the 7-day window. Dashed grid, hidden toolbar/axis borders, top-right legend.
- **Station Status**: **donut** with a 74% hole rendering live **uptime %** (`online/total`) as the center total label; paired with a legend stack (Online / Offline / Total Connectors).
- **Palette** (`PALETTE` const): primary `#2E7D32`, bright `#00C853`, light `#66BB6A`, blue accent `#2196F3`, orange accent `#FF9800`. Donut uses bright-green for online, orange for offline; trend series use bright-green + blue.

## Widget → redux data mapping
Selector shape and the four dispatches (`fetchOverview`, `fetchSessionTrends({period:'7d'})`, `fetchTopStations({limit:5})`, `fetchLiveSessions`) are **unchanged**.

| Widget | Data source |
|---|---|
| Total Stations card | `overview.stations.total` (+ online/offline subtitle) |
| Active Sessions card | `overview.activeSessions` (+ `overview.connectors` subtitle) |
| Today's Energy card | `overview.today.energy` (kWh) |
| Today's Revenue card | `overview.today.revenue` (₹, "QR & wallet top-ups" caption) |
| Session Trends area chart | `sessionTrends[].period / .sessions / .energy` |
| Station Status donut + legend | `overview.stations.{online,offline,total}`, `overview.connectors` |
| Live Sessions list | `liveSessions[].{id, station.name, driver.name, energyDelivered, status}` via `StatusChip` |
| Top Stations list | `topStations[].{stationId, station.name, sessionCount, totalRevenue}` — revenue normalized against `maxRevenue` to drive a `LinearProgress` bar; top-3 highlighted |

`fetchRecentActivity` is exported by the slice but is **not** dispatched (no widget consumes it yet — left for a future activity feed).

## Responsive behavior
Every row uses `xs/sm/md` breakpoints so the page reflows from single-column (mobile) → 2-column (tablet) → full multi-column (desktop). Header uses `PageHeader`'s built-in `flexWrap`; the trend-card header row also wraps. Charts are width-responsive via ApexCharts. Loading spinner behavior preserved (`loading && !overview` → centered `CircularProgress`, now green-tinted).

## Follow-ups / backend endpoints still needed
- **Recent Activity feed**: `fetchRecentActivity` already exists in the slice but has no UI/endpoint surfaced — a future "Recent Activity" timeline card could consume it.
- **Trend deltas for KPIs**: `StatsCard` supports a `trend`/`trendLabel` prop. The overview endpoint returns no comparison figures, so trend arrows were omitted — add yesterday/last-week deltas to the overview payload to light these up.
- **Station geo data**: a map widget would need lat/lng on the stations endpoint (`sttn_lst_t.ltde_nbr/lngtde_nbr`).
- **Live refresh**: data is fetched once on mount — add polling or a websocket to make "Live Sessions" truly live.
- **Backend gap**: the admin dashboard endpoints (`/dashboard/overview`, `/top-stations`, `/live-sessions`, `/station-status`) do not yet exist in djt-app — see `docs/DJT_ADMIN_INTEGRATION.md` §6–§7 for the SQL and endpoint plan.
